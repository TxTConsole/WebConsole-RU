## 🚀 **Что такое WebConsole?**

**WebConsole** — это мощная веб-панель для администрирования Minecraft серверов на ядрах **Paper**, **Spigot** или **Purpur**. Она позволяет полностью управлять сервером через браузер с вашего компьютера.

Забудьте про SSH или RDP только для того, чтобы выполнить простую команду или посмотреть консоль. С WebConsole всё доступно через современный интерфейс в стиле «неоновое стекло» с обновлением в реальном времени.

**[Скачать плагин SpigotRU](https://spigotmc.ru/resources/webconsole.5257/)**

## ✨ **Основные возможности**

### 🖥️ **Консоль в реальном времени**
- Прямой вывод консоли с цветовой разметкой уровней логирования (INFO, WARN, ERROR)
- Поиск и фильтрация логов по уровню или ключевым словам
- Мгновенное выполнение команд
- История команд и справочник доступных команд (кнопка «?» — скопировать и вставить команду в консоль)
- Автопрокрутка и скачивание логов

### 👥 **Управление игроками**
- Просмотр онлайн-игроков с пингом и временем игры
- Кик, бан, мут игроков прямо из интерфейса
- Возможность бана по IP-адресу
- Просмотр и управление наказанными игроками (разбан, размут)
- Управление вайтлистом (белым списком) и операторами (OP)

### 📁 **Файловый менеджер (БЕТА)**
- Просмотр, редактирование и удаление файлов сервера прямо в браузере
- Подсветка синтаксиса для YAML, JSON, properties, Java и других форматов (на базе CodeMirror)
- Загрузка файлов через перетаскивание или выбор файла
- Создание новых файлов и папок
- Скачивание любых файлов с сервера
- Удобная навигация по пути

> ⚠️ **Примечание:** Файловый менеджер находится в стадии бета-тестирования. Возможны ошибки и недочёты. Пожалуйста, сообщайте о найденных проблемах.

### 🧩 **Менеджер плагинов (БЕТА)**
- Поиск плагинов прямо из интерфейса через Modrinth
- Автоматическая проверка совместимости с версией вашего сервера
- Установка плагинов в один клик
- Переустановка или удаление существующих плагинов
- Отображается автор, количество загрузок и рейтинг

> ⚠️ **Примечание:** Менеджер плагинов находится в стадии бета-тестирования. Возможны ошибки при установке или поиске. Рекомендуется делать резервные копии перед установкой новых плагинов.

### ⚙️ **Настройки сервера**
- Визуальное редактирование `server.properties` (порт, макс. игроков, дистанция прорисовки и др.)
- Переключение настроек: online-mode, PvP, полёты, командные блоки
- Управление вайтлистом с мгновенным применением
- Управление списком операторов (OP)
- Выбор сложности (Мирная, Лёгкая, Нормальная, Сложная)

> ⚠️ **Важно:** После изменения настроек сервера требуется перезагрузка сервера для применения изменений.

### 🔐 **Безопасность**
- Авторизация с поддержкой нескольких пользователей
- Токены сессий с настраиваемым временем жизни
- Защита API с помощью Bearer-токенов

### 🎨 **Современный интерфейс**
- Дизайн в стиле «неоновое стекло» с анимированным фоном
- Полная настройка акцентного цвета и цветов логов
- Адаптивный интерфейс, оптимизированный для работы на ПК
- Подсказки с полезной информацией на каждом элементе

### 🌍 **Поддержка нескольких языков (БЕТА)**
- **Русский**
- **English**

> ⚠️ **Примечание:** Поддержка языков находится в стадии бета-тестирования. Возможны ошибки перевода или некорректное отображение текста. Если вы заметили ошибку, сообщите автору.

### 🔊 **Настраиваемые уведомления**
- Всплывающие уведомления в стиле рабочего стола
- Звуковые оповещения (можно загрузить свой звуковой файл)
- Выбор позиции уведомлений (верх/низ, лево/право)
- Кнопка тестового уведомления

## 🔧 **Установка**

1. Скачайте последнюю версию `WebConsole.jar`
2. Поместите файл в папку `plugins/` вашего сервера
3. Перезапустите сервер
4. Откройте браузер на компьютере и перейдите по адресу `http://ip-вашего-сервера:8080`
5. Войдите с учётными данными по умолчанию:
   - Логин: `admin`
   - Пароль: `admin123`
6. **Обязательно смените пароль** в файле `plugins/WebConsole/config.yml`

## 📋 **Конфигурация (`config.yml`)**

```yaml
# ==========================================
# WebConsole Configuration v1.0.0
# ==========================================

server:
  # Порт для веб-интерфейса
  # После изменения требуется перезагрузка сервера
  port: 8080

security:
  # Включить авторизацию (true/false)
  # Если false, доступ будет открыт без пароля (не рекомендуется!)
  enabled: true

  # Время жизни сессии в часах (для тех, кто поставил галочку "Запомнить меня")
  # По истечении времени потребуется повторный вход
  session-timeout-hours: 24

  # Список пользователей (логин: "пароль")
  # Вы можете добавить любое количество пользователей
  users:
    admin: "admin123"
    moderator: "mod123"
    # user: "password" - пример добавления нового пользователя

  # Разрешить выполнение команд из веб-консоли (true/false)
  # Если false, поле ввода команд будет отключено
  allow-commands: true

console:
  # Максимальное количество строк, хранимых в памяти
  # Большее значение потребляет больше оперативной памяти
  max-history-limit: 500

  # Формат времени в логах
  time-format: "HH:mm:ss"

# Системные сообщения
messages:
  server-started: "WebConsole запущен на порту"
  access-denied: "У вас нет прав для выполнения этой команды"
```

## 💡 **Почему WebConsole?**

- **Не требует внешних зависимостей** — работает на встроенном HTTP-сервере Java
- **Работает «из коробки»** — нулевая настройка
- **Безопасность** — токен-авторизация с таймаутом сессии
- **Лёгкость** — минимальное влияние на производительность сервера
- **Открытый исходный код** — хостится на GitHub под лицензией GPL-3.0

## ❓ **Часто задаваемые вопросы**

**Вопрос: Можно ли использовать плагин на публичном сервере?**  
Да, но обязательно смените пароль по умолчанию и по возможности используйте HTTPS.

**Вопрос: Работает ли с BungeeCord / Velocity?**  
На данный момент плагин разработан только для Bukkit/Spigot/Paper серверов. Поддержка прокси может появиться в будущих обновлениях.

**Вопрос: Как перезагрузить сервер?**  
Нажмите кнопку «Рестарт» в верхней панели. Сервер сохранит все данные и перезагрузится.
